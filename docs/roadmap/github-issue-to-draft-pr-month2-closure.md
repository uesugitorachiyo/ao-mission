# GitHub Issue To Draft PR Month 2 Closure

Month 2 defines the authenticity and reproduction gate for the GitHub
issue-to-draft-PR roadmap.

The required Month 2 state is:

- Issue text and repository content remain untrusted inputs.
- Repository identity, upstream URL, default branch, and target commit are
  recorded before reproduction.
- Acquisition uses hooks-disabled, safe Git configuration, controlled remotes,
  sanitized environment, and read-only command planning.
- Authentic-bug classification requires failing pre-patch reproduction evidence.
- Flaky cases require repeated-run measurement; uncertainty is not promoted to
  an authentic-bug claim.
- Non-bug, policy-blocked, security-sensitive, prompt-injection, and
  cannot-reproduce cases avoid mutation and do not enter public repair.
- Command readback shows terminal states and the next safe action.
- Sentinel blocks false authenticity, public security repair, provider-pilot,
  release, and RSI overclaims.

The Month 2 truth set contains 13 fixtures: 2 authentic-bug fixtures and 11
non-bug or blocked fixtures. Precision and recall thresholds are met by the
fixture evidence. Feature-generated pull requests remain draft-only by policy;
they are not merged, marked ready, or approved by this roadmap.

RSI remains denied. External beta is not launched. Promotion is not requested
or granted. No release, tag, upload, or deployment is selected by Month 2.

Next action after Month 2 closure: continue to Month 3 isolated repair,
verification, rollback, and replay.
